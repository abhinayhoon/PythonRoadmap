Loops are used when you want to repeat an action multiple times without writing the same code again and again.

Why loops exist

Imagine printing numbers from 1 to 100. Without loops, you would need 100 print statements. Loops save time, reduce mistakes, and make programs scalable.

The for loop

A for loop is used when you know **how many times** you want to repeat something. 


for i in range(5):
    print(i)

Explanation:
range(5) generates numbers from 0 to 4. Each number is stored in i one by one. The loop runs 5 times.

Loop starting from 1


for i in range(1, 6):
    print(i)

Here, range(1, 6) starts at 1 and stops before 6.

The while loop

A while loop runs **as long as a condition is True**. It is used when you don‚Äôt know in advance how many times the loop will run. 


count = 1

while count <= 5:
    print(count)
    count += 1

Explanation:
The loop checks the condition before every run. If the condition becomes False, the loop stops.

Important warning ‚ö†Ô∏è

If you forget to update the condition inside a while loop, it may run forever. This is called an infinite loop.

Mental model

for loop: Repeat a fixed number of times
while loop: Repeat until a condition changes

Mini practice

  • Print numbers from 1 to 10
  • Print only even numbers
  • Try changing loop ranges

Why return is powerful

Functions that return values can be used in calculations, conditions, loops, and even inside other functions.

Until now, one variable stored one value. But real programs often deal with multiple values of the same type. Python solves this using lists.

What is a list?

A list is a collection of items stored inside a single variable. Lists can store numbers, text, or even other lists.

Creating a list


fruits = ["apple", "banana", "mango"]
print(fruits)

Items in a list are written inside square brackets [] and separated by commas.

Accessing list elements (indexing)


fruits = ["apple", "banana", "mango"]
print(fruits[0])
print(fruits[1])

Important: List indexing starts from 0, not 1.

Changing list values


fruits[1] = "orange"
print(fruits)

Lists are mutable, meaning their values can be changed.

Common list operations


numbers = [1, 2, 3]

numbers.append(4)
numbers.remove(2)
print(len(numbers))

append() adds an item, remove() deletes an item, len() gives total number of items.

Using loops with lists


fruits = ["apple", "banana", "mango"]

for fruit in fruits:
    print(fruit)

Python automatically goes through each item in the list.

Mental model

A list is like a row of boxes. Each box has a position (index) and a value inside it.

Mini practice

  • Create a list of 5 numbers
  • Print the first and last number
  • Add a new number to the list
  • Loop through the list and print all values

Lists are flexible, but sometimes you want data that should not change or collections that should not have duplicates. Python provides tuples and sets for this.

Tuples (immutable collections)

A tuple is like a list, but it cannot be changed after creation. This makes tuples safer for fixed data. 


coordinates = (10, 20)
print(coordinates[0])
print(coordinates[1])

Tuples use parentheses (). You can read values, but you cannot modify them.

Why use tuples?

  • Data should not change (days of week, coordinates)
  • More memory-efficient than lists
  • Safer in large programs

Attempting to modify a tuple (error example)


days = ("Mon", "Tue", "Wed")
days[0] = "Sunday"   # This will cause an error

Sets (unique collections)

A set stores unique values only. Duplicate values are automatically removed. 


numbers = {1, 2, 3, 3, 4}

Even though 3 was added twice, it appears only once.

Common set operations


numbers = {1, 2, 3}

numbers.remove(2)
print(numbers)

Sets are useful for checking membership quickly.

Checking membership


users = {"Alice", "Bob", "Charlie"}

if "Bob" in users:
    print("User exists")

Mental model

List: ordered, changeable, allows duplicates
Tuple: ordered, unchangeable
Set: unordered, unique values only

Until now, all data existed only while the program was running. Once the program stopped, the data was lost. File handling allows programs to store data permanently.

What is file handling?

File handling means reading data from files and writing data to files. Files are used to store information like logs, user data, reports, and results.

Opening a file


file = open("data.txt", "r")

Explanation:
"data.txt" is the file name.
"r" means read mode.

Reading from a file


file = open("data.txt", "r")
content = file.read()
print(content)
file.close()

Always close the file after reading to free system resources.

Writing to a file


file = open("data.txt", "w")
file.write("Hello, this is written to a file.")
file.close()

Important: "w" mode overwrites the file if it already exists.

Appending to a file


file = open("data.txt", "a")
file.write("\nThis line is added later.")
file.close()

"a" mode adds new content without deleting existing data.

The recommended way: using with


with open("data.txt", "r") as file:
    content = file.read()
    print(content)

Using with automatically closes the file. This is safer and cleaner.

File modes summary

  • "r" ‚Üí read
  • "w" ‚Üí write (overwrite)
  • "a" ‚Üí append

Mental model

A file is like a notebook on disk. Your program opens it, reads or writes, and then closes it.

Common beginner mistakes

  • Using "w" instead of "a"
  • Trying to read a file that doesn‚Äôt exist

Until now, when an error occurred, your program stopped immediately. This is bad for real applications. Error handling allows your program to handle mistakes gracefully instead of crashing.

What is an error?

An error occurs when Python encounters something it cannot understand or cannot execute. Examples include wrong input, missing files, or invalid operations.

Example of a program crash


number = int(input("Enter a number: "))
print(10 / number)

If the user enters 0 or text, the program will crash with an error.

Handling errors using try and except


try:
    number = int(input("Enter a number: "))
    print(10 / number)
except:
    print("Something went wrong")

Explanation:
Python tries to run the code inside try. If an error happens, it jumps to except instead of crashing.

Handling specific errors


try:
    number = int(input("Enter a number: "))
    print(10 / number)
except ValueError:
    print("Please enter a valid number")
except ZeroDivisionError:
    print("Division by zero is not allowed")

Handling specific errors gives better messages and better control.

Using else with try


try:
    number = int(input("Enter a number: "))
except ValueError:
    print("Invalid input")
else:
    print("You entered:", number)

The else block runs only if no error occurs.

Using finally


try:
    file = open("data.txt", "r")
    print(file.read())
except FileNotFoundError:
    print("File not found")
finally:
    print("Program finished")

finally always runs, whether an error occurred or not.

Mental model

Think of try as: ‚ÄúAttempt this safely.‚Äù If something breaks, Python uses except instead of stopping everything.

Common beginner mistakes

  • Using a broad except without specifying error types
  • Hiding errors instead of fixing logic
  • Overusing try/except everywhere

Mini practice

  • Ask the user for two numbers
  • Handle invalid input
  • Handle division by zero
  • Print a friendly message instead of crashing

Modern applications exchange data using JSON. APIs, configuration files, and databases commonly use JSON format. Python provides built-in support for working with JSON easily.

What is JSON?

JSON stands for JavaScript Object Notation. It is a lightweight format used to store and transfer data. JSON looks very similar to Python dictionaries.

Example of JSON data


{
  "name": "Alex",
  "age": 18,
  "is_student": true
}

Keys are strings, values can be numbers, strings, booleans, lists, or other JSON objects.

The json module


import json

Python‚Äôs json module allows converting between Python data and JSON data.

Converting Python to JSON (serialization)


import json

data = {
    "name": "Alex",
    "age": 18,
    "skills": ["Python", "HTML"]
}

json_data = json.dumps(data)
print(json_data)

json.dumps() converts Python objects into a JSON-formatted string.

Converting JSON to Python (deserialization)


import json

json_text = '{"name": "Alex", "age": 18}'
python_data = json.loads(json_text)

print(python_data["name"])

json.loads() converts JSON text into a Python dictionary.

Reading JSON from a file


import json

with open("data.json", "r") as file:
    data = json.load(file)

print(data)

Writing JSON to a file


import json

data = {
    "project": "Python Roadmap",
    "day": 14
}

with open("data.json", "w") as file:
    json.dump(data, file)

JSON files are commonly used for settings and saved data.

Mental model

JSON is a universal language for data. Python converts dictionaries into JSON and JSON back into dictionaries.

Common beginner mistakes

  • Confusing load with loads

As projects grow, they start depending on external libraries. Different projects may need different versions of the same library. This is where virtual environments become essential.

What is pip?

pip is Python‚Äôs package manager. It is used to install, update, and remove external libraries. 


pip install requests

This command downloads and installs the requests library.

Why virtual environments are needed

Installing packages globally can cause conflicts between projects. Virtual environments isolate dependencies per project.

Creating a virtual environment


python -m venv venv

This creates a folder named venv containing an isolated Python environment.

Activating the virtual environment

On Windows:


venv\Scripts\activate

On macOS / Linux:


source venv/bin/activate

Once activated, any package you install belongs only to this project.

Installing packages inside venv


pip install flask

The package is installed inside the virtual environment, not globally.

Freezing dependencies


pip freeze > requirements.txt

This creates a file listing all installed packages. It allows others to recreate the same environment.

Installing from requirements file


pip install -r requirements.txt

Deactivating the virtual environment


deactivate

Mental model

A virtual environment is a sealed room. Everything installed inside stays inside, and does not affect other projects.

Common beginner mistakes

  • Forgetting to activate the virtual environment
  • Installing packages globally by mistake
  • Not using requirements.txt

Mini practice

Until now, we wrote programs using functions and variables. This works, but as programs grow, managing everything becomes hard. Object-Oriented Programming (OOP) helps structure large programs in a clean and logical way.

What is OOP?

OOP is a programming style where we model real-world entities as objects. Each object has data and behavior.

What is a class?

A class is a blueprint. It defines what data and behavior an object will have. Objects are created from classes.

Creating a class


class Student:

Creating an object

In Day 17, you learned how to create classes and objects. Now we go one level deeper. Inheritance and Polymorphism allow you to reuse code and write flexible programs.

What is inheritance?

Inheritance allows one class to reuse the properties and methods of another class. The new class is called a child, and the original class is called a parent.

Real-life analogy

A Car and a Bike are both Vehicles. They share common features like speed and fuel, but they also have their own unique behaviors.

Creating a parent class


class Vehicle:
    def __init__(self, brand):
        self.brand = brand

    def start(self):
        print("Vehicle started")

Vehicle is a parent class. It defines behavior common to all vehicles.

Creating a child class


class Car(Vehicle):
    def drive(self):
        print("Car is driving")

The Car class inherits from Vehicle. It automatically gets access to the start() method.

Using inherited methods


car = Car("Toyota")
car.start()
car.drive()

Even though start() is not defined in Car, it can still be used because of inheritance.

Method overriding

A child class can redefine a method from the parent class. This is called method overriding. 


class Bike(Vehicle):
    def start(self):
        print("Bike started with kick")

The Bike class replaces the parent's start() behavior with its own.

What is polymorphism?

Polymorphism means same method name, different behavior. Different objects respond differently to the same method call.

Polymorphism example


vehicles = [Car("Honda"), Bike("Yamaha")]

for v in vehicles:
    v.start()

Even though the method name is the same, Python calls the correct version based on the object type.

Why this matters

  • Less repeated code
  • Cleaner structure
  • Easier extension of programs
  • Core concept behind frameworks

Mental model

Inheritance = sharing features
Polymorphism = same action, different behavior
Together they make code flexible and powerful.

Common beginner mistakes

  • Overusing inheritance when not needed
  • Forgetting the parent class name in brackets
  • Confusing overriding with overloading

Mini practice

  • Create a parent class Animal
  • Create child classes Dog and Cat
  • Override a method like sound()
  • Call the method in a loop to see polymorphism

Files are useful, but when data grows, managing it becomes slow and messy. Databases solve this problem by storing data in a structured, searchable, and efficient way.

What is a database?

A database is an organized collection of data. It allows fast storage, retrieval, updating, and deletion of information.

Why SQLite?

SQLite is a lightweight, built-in database in Python. It does not require installation or a server. Perfect for beginners and small applications.

Importing SQLite in Python


import sqlite3

Creating a database connection


import sqlite3

conn = sqlite3.connect("students.db")

This creates a database file named students.db. If it doesn‚Äôt exist, Python creates it automatically.

Creating a table


cursor = conn.cursor()

cursor.execute("""
CREATE TABLE students (
    id INTEGER PRIMARY KEY,
    name TEXT,
    age INTEGER
)
""")

conn.commit()

A table stores data in rows and columns, similar to a spreadsheet.

Inserting data into the table


cursor.execute(
    "INSERT INTO students (name, age) VALUES (?, ?)",
    ("Alex", 18)
)

conn.commit()

The ? placeholders prevent SQL injection and make code safer.

Reading data from the database


cursor.execute("SELECT * FROM students")
rows = cursor.fetchall()

for row in rows:
    print(row)

Each row is returned as a tuple.

Updating data


cursor.execute(
    "UPDATE students SET age = ? WHERE name = ?",
    (19, "Alex")
)

conn.commit()

Deleting data


cursor.execute(
    "DELETE FROM students WHERE name = ?",
    ("Alex",)
)

conn.commit()

Closing the database


conn.close()

Always close the database connection when you are done.

Mental model

A database is like an organized notebook: tables are pages, rows are entries, and columns define structure.

Common beginner mistakes

  • Forgetting to commit changes
  • Not closing the database connection
  • Writing SQL directly with user input

  • Add multiple book records
  • Fetch and display all books
  • Update and delete a record

import sqlite3

conn = sqlite3.connect("students.db")

CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

)
""")

conn.commit()

Step 2: Function to add a student


def add_student(name, age):
    cursor.execute(
        "INSERT INTO students (name, age) VALUES (?, ?)",
        (name, age)
    )
    conn.commit()

Step 3: Function to view students


def view_students():
    cursor.execute("SELECT * FROM students")
    students = cursor.fetchall()
    for student in students:
        print(student)

Step 4: Handling user input safely


try:
    age = int(input("Enter age: "))

Step 5: Menu-driven loop


while True:
    print("1. Add Student")

    choice = input("Choose an option: ")

    if choice == "1":
        name = input("Enter name: ")

    elif choice == "2":
